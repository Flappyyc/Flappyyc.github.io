「bzoj1055」玩具取名

题面

Description

　　某人有一套玩具，并想法给玩具命名。首先他选择WING四个字母中的任意一个字母作为玩具的基本名字。然后他会根据自己的喜好，将名字中任意一个字母用“WING”中任意两个字母代替，使得自己的名字能够扩充得很长。
　　现在，他想请你猜猜某一个很长的名字，最初可能是由哪几个字母变形过来的。

Input

　　第一行四个整数W、I、N、G。表示每一个字母能由几种两个字母所替代。接下来W行，每行两个字母,表示W可以用这两个字母替代。接下来I行，每行两个字母,表示I可以用这两个字母替代。接下来N行，每行两个字母,表示N可以用这两个字母替代。接下来G行，每行两个字母,表示G可以用这两个字母替代。最后一行一个长度不超过Len的字符串。表示这个玩具的名字。

Output

　　　一行字符串，该名字可能由哪些字母变形而得到。（按照WING的顺序输出）如果给的名字不能由任何一个字母变形而得到则输出“The name is wrong!”

Sample Input

　　1 1 1 1
　　II
　　WW
　　WW
　　IG
　　IIII

Sample Output

　　IN

HINT

　　W可以变成II所以IIII可以缩成WW IN均能变成WW所以WW又可以缩成I或者N 所以最终答案应该按照“WING”的顺序

输出

IN

[数据范围]

　　100%数据满足Len<=200，W、I、N、G<=16

题解

这是一道比较容易的区间dp。首先，自始至终字母一定是只有W I N G这四个。
定义f[i,j,k]代表，从i到j的字符串可以由一个第k种字母开始并形成。
当k可变成ab并且(f[i][p][a] & f[p+1][j][b]) (i<=p<j)时，f[i][j][k]=true。
由于方程的形式，此题用记忆化搜素。注意，区间dp经常会用到记忆化搜索。

dp小结

Posted on 2018-04-18

动态规划

注意点：

可以考虑从一个已知状态应该更新哪些后续状态，也可以考虑如何计算出一个状态（一个状态由哪些状态来得到）。

两种思维方式都可以考虑，正向思维和逆向思维，根据当时情况来决定。有时两种选择的差异会体现出来。

动态规划优化可以从时间和空间两方面上来进行。

空间上，一般利用滚动数组可以减少空间的维数。
时间上，可以仔细观察结构，注意决策集合的范围随状态变化情况。
如果决策集合中元素只增不见，就可以把时间复杂度降一个量级。

每个子问题求解过程称为一个阶段。求解线性dp，先确定阶段，定下状态时，必要的附加信息也可以加入。确定状态以后，要尽可能缩小维度，寻找各维度之间的关系，排除不需要的维度。

有时可以通过额外算法确定dp顺序（比如先给元素排序），也可以在状态空间中利用等效手法对状态进行缩放。

动态规划中循环的顺序是固定的，不要搞混。从外层到内层分别是阶段——阶段——决策，不能颠倒。

动态规划三要素：阶段 状态 决策

动态规划三前提：无后效性 最优子结构 子问题重叠

任意选择一个位置断开，并复制成一条2倍长度的链，是动态规划中解决环形问题常用的方法。

背包

1.01背包
最简单的模型，O(nv)的时间复杂度。
但是，可以在空间复杂度上做文章。最基本的算法，空间复杂度和时间复杂度一样，也是O(nv)。但是，由于有一维固定地从i-1转移到i，所以，可以利用滚动数组。这样，空间复杂度就变成了O(2*n)=O(n)。继续观察，发现，当里面一层循环倒序进行，可以不用滚动数组，每次计算利用到的数组都是没有修改过的。这样，就变成了f[n]就完成了01背包的所有操作。

2.完全背包
完全背包就是每个物品有无数个，其他与01背包一样。按照原始的算法，完全背包的时间复杂度比01背包要多一维。但是，仔细思考，发现，01背包是用的是倒序循环，而完全背包只要改成正序循环，就完全符合条件了，这样每个物品可以用无数次的特点就体现在了程序中，这样的时间复杂度就与01背包相同了，都是O(nv)。

3.多重背包
最朴素的做法，把一定数量的同一个物品拆分成没有关系的单个物品。这样子时间复杂度会继续上升，在题目中就不太适用了。可以利用两种方法，二进制拆分法和单调队列进行优化。二进制拆分法比单调队列多了一个log，这里就先介绍二进制拆分法。
设Ci表示第i个物品有Ci件。令p为使2^0+2^1+2^2+……+2^p<=Ci的最大整数。
记Ri为Ci-2^0+2^1+2^2+……+2^p
2^0+2^1+2^2+……+2^p+2^(p+1)>Ci 所以2^(p+1)>Ri，所以2^0，2^1，2^2，… ，2^p取若干个相加可以表示0-Ri的所有整数。
继续来看，从2^0，2^1，2^2，……，2^p，Ri中取若干个相加可以表示Ri至Ri+2^(p+1)-1的任意整数。根据定义，Ri+2^(p+1)-1=Ci。所以2^0，2^1，2^2，……，2^p，Ri中取若干个相加就可以表示0-Ci之间的所有整数。
于是，把数量为Ci的i物品拆为p+2件，2^0Vi,2^1Vi,2^2Vi…….2^pVi,Ri*Vi，接下来就变成01背包了。复杂度相较于01背包多了一个log，大部分时间下也是可行的。

4.分组背包
把每一组作为阶段，朴素的进行dp。剩余过程与01背包一样，循环记得倒序。

区间dp

区间dp通常来说已区间的长度作为阶段。这样的话，根据注意点的第5条，len这一层的循环一定要在循环的最外层，这是需要小心的。

对于方案计数类的动态规划问题，一个状态的各决策之间满足加法原理，每个决策划分的几个子状态满足乘法原理。一个状态的各决策之间一定要有互斥性，才能保证动态规划正确，做到不重不漏。

树形dp

1.背包类树形dp
背包类树形dp一般是树形dp与背包的结合。
把节点编号作为背包的阶段，当前背包内物品体积作为第二维状态。
状态转移时，也类似于分组dp。

2.二次扫描与换根法
二次扫描与换根法一般说的是
某些无根树树形dp的朴素算法需要以每个节点为根做一次dp，并最后求出最值。
而二次扫描换根法只需找一个节点，做一次dp。从该点开始，以该点为根，做一次全树遍历，可以通过相邻两节点之间的已知数据进行计算，计算出它所有子节点的值。这样就可以降低时间复杂度。

3.环形与后效性处理
对于环形结构上的动态规划问题，可以采用2次dp。一次是把环拆成链，做一个线性dp，第二次dp是通过一些条件和赋值，来强制这条链的首尾相连。

还有另一种方法，即把一条长度为n的环断开，在末尾再接上一段，使其长度为2
n。这样，也可以把问题转变为线性问题。

关于后效性处理，动态规划的一个前提是无后效性，但是有的方程是具有后效性的。这个时候，就要观察方程，并利用合适的方法，比如高斯消元。注意，在很多数学期望的动态规划里，都采用倒推的方式，（正推还涉及概率问题）

状态压缩dp

状态压缩dp，进行动态规划时要注意转移方式，需要没有缺漏。十进制数在进行位运算是需要注意。

倍增优化dp

与其它的倍增一样，第一部分为预处理，第二部分为拼凑，计算若干与2的幂次有关的状态。

数据结构优化dp
例如方程f[bi]=min{f[x]}+ci。方程需要在线求最值和更新，可以联想到线段树。
ai-1<=x<bi

观察方程类型，根据需要进行的操作，来选择合适的数据结构。

无论dp限制条件是多是少，都尽量对其进行分离。可以把最外层设为定值，再在内层进行操作。
